# SQL Server 2022/Azure Exact Rowset Extraction Status

Search key: SB_REFERENCE_REFERENCE_SQLSERVER_2022_AZURE_EXACT_ROWSET_STATUS

Scope correction: SQL Server is not emulated by ScratchBird. Exact SQL Server rowset extraction is not required for ScratchBird runtime catalog seeding and must not be used as runtime authority. This file is retained only as a superseded reference artifact.

## Status

The copied private reference material defines SQL Server/Azure catalog objects, column names, broad semantics, and built-in behavior groups. It does not yet contain exact default rowsets from a newly-created SQL Server 2022 database or Azure SQL database.

The SQL Server capability-reference pack is not a catalog-seeding target. Future work must create `CommercialCapabilityReferenceRecord` rows and, where needed, an outbound C++ connectivity UDR profile.

## Generated values

The final seed pack must classify each generated value as one of:

- stable reference alias over ScratchBird UUID
- deterministic value generated by a SQL Server capability-reference profile when retained for comparison
- runtime value generated per database open
- runtime value generated per connection
- runtime value generated per transaction
- report-only value derived from `sys.metrics.*`
- report-only value derived from `cluster.sys.metrics.*` when and only when the database is joined to a cluster
- forbidden because it would expose a raw secret, private key, credential, OS path, or host-only implementation detail

Any generated value without one of these classifications must raise `REFERENCE.CATALOG_SEED.RUNTIME_VALUE_UNCLASSIFIED`.

## Extraction requirements

The final extraction must capture:

- SQL Server 2022 default database catalog rowsets.
- Azure SQL default database catalog rowsets where they differ.
- Column order and exact reference datatype for every catalog surface.
- Nullability and default display value for every column.
- Visibility differences for normal user, database owner, sysadmin-equivalent, and denied principal.
- Metadata visibility behavior when `VIEW DEFINITION` is absent.
- Redaction behavior for encrypted, credential, login, and security-sensitive fields.
- Rowset deltas after creating a table, view, procedure, trigger, index, sequence, user-defined type, schema, role, principal, permission, temporal table, ledger object, graph object, and memory-optimized object where supported by the profile.

## ScratchBird mapping rule

The exact reference rowset must never become internal authority. ScratchBird internal authority remains UUID-backed catalog, security, metrics, and resolver rows. SQL Server rowsets are compatibility projections.

SQL Server object IDs, principal IDs, database IDs, schema IDs, type IDs, file IDs, and index IDs are reference values only. They must never replace ScratchBird UUID identity.
