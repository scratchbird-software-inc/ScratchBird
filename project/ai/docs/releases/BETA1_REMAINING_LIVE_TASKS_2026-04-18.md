## 1. Runtime Profile

Canonical runtime metadata is supplied by policy:

- `SCRATCHBIRD_AI_LIVE_NATIVE_RUNTIME_ENV_PATH`
- `SCRATCHBIRD_AI_CONNECTION_PROFILE_PATH`

Endpoint state used by the rerun recipe below:

- Native ScratchBird listener details are read from the configured profile.
- Database, user, and secret values are deployment configuration and must not be
  committed in this repository.
- Reference emulation listeners may be profile-defined but are outside this
  component's supported AI surface.

Only the native listener is part of the AI repo live-certification path.
Reference-emulation listener startup remains outside this repository's supported AI
surface.

## 2. Completed Live Tasks

1. Bring up the refreshed ScratchBird test environment that matches the
   completed core AI surface packet.
2. Run live HTTP smoke against the target.
3. Run live native conformance capture against the target.
4. Regenerate the tracked conformance artifacts on the same commit after the
   live run.
5. Re-run the evidence-gate validator on the regenerated artifacts.
6. Record the remaining runtime-mode-specific follow-up for `manager_proxy`,
   `local_ipc`, and `embedded_local_only` if those modes are exercised in a
   future refreshed environment.

## 3. Canonical Command Sequence

Ubuntu/Linux reviewer path:

```bash
export PYTHONPATH=src
export SCRATCHBIRD_AI_ADAPTER_MODE=http
export SCRATCHBIRD_AI_LIVE_NATIVE_ENABLED=1
export SCRATCHBIRD_AI_LIVE_NATIVE_LAUNCH_BRIDGE=1
export SCRATCHBIRD_AI_LIVE_NATIVE_RUNTIME_ENV_PATH="${SCRATCHBIRD_AI_LIVE_NATIVE_RUNTIME_ENV_PATH:-$HOME/.scratchbird/static-example/profiles/runtime.env}"
python3 tools/run_live_native_conformance.py \
  --scratchbird-server-version current-shared-node-2026-04-20 \
  --parser-compiler-version current-v3-prebuild \
  --test-dataset-version shared-main \
  --seed-or-fixture-version shared-node \
  --covered-profile mcp_local_v0 \
  --covered-profile mcp_remote_v0 \
  --covered-profile streaming_async_v0 \
  --covered-profile retrieval_ingest_v0
python3 tools/generate_ai_conformance_artifacts.py --repo-root .
python3 tools/validate_evidence_gates.py --repo-root . --spec docs/releases/EARLY_BETA_CONFORMANCE_GATES.md
```

## 4. Acceptance Outcomes

The direct-listener Beta 1 live closeout is complete when:

- the live smoke passes
- the live native conformance harness produces current artifacts
- regenerated tracked artifacts remain internally consistent on one commit
- the evidence-gate validator passes
- no remaining blockers exist outside explicitly draft, future-hardening, or
  out-of-scope surfaces

If the live harness reports engine-endpoint diagnostics such as
`base_socket=missing` or `parser_socket=stale_or_not_listening`, treat that as
a ScratchBird runtime blocker for the refreshed environment and do not refresh
the AI release packet until the server/listener IPC path is corrected.
